О проекте

Приложение "Place in text" собирает по запросу пользователя данные о статьях, опубликованных когда-либо на новостном сайте Lenta.ru, и отображает геообъекты, обнаруженные в тексте статей, на карте, а также выводит их координаты.

Как использовать:

  1. Во вкладке "Фильтр" выбрать временной диапазон запроса;
  2. (Опциально) Выбрать, будет результат упорядочен по алфавитному порядку геообъектов или по датам;
  3. Нажать кнопку "Поиск" и больше не совершать каких-либо действий, загрузка записей с базы данных может занять какое-то время;
  4. Во вкладке "Результат запроса" можно выбрать интересующую запись и нажать на нее, таким образом текст статьи загрузится во вкладку "Текст статьи", геометка обнаруженного объекта отобразится в центре карты (вкладка "Карта"), а также во вкладке "Координаты объекта" отобразятся координаты объекта;
  5. Во вкладке "Карта" можно повзаимодействовать с интерактивной картой: посмотреть на положение метки на карте, покрутить карту и изменить ее масштаб.
Примечания:
  1. По умолчанию приложение сортирует статьи по датам, но, поставив галочку "Упорядочить по местоположению?", можно отсортировать статьи по алфавитному порядку геообъектов (при этом в рамках одного геообъекта также будет сортировка по датам);
  2. API Nominatim работает так, что на принятый им запрос он вернет самый первый результат, даже если он не соответсвует оригинальной статье;
  3. Некоторые из геообъектов могли быть обнаружены неверно, или использованное API геокодера может не найти выбранный геообъект (или найти не тот объект). На это влияет цепочка несовершенств:
    • Несовершенство редактирования оригинальных статей;
    • Несовершенство собранного датасета;
    • Несовершенство использованной ML-модели Natasha (точность составила около 95%);
    • Несовершенство использованного API геокодера Nominatim (ручная отладка запросов показала, что некоторые записи устаревшие, некоторые записи не распознаются на русском языке, а некоторые записи не соответствуют целевым).
    С ними на данном этапе существования проекта ничего не поделать, но можно хотя бы ознакомиться с текстом оригинальной статьи и выяснить, на каком звене вышеописанной цепочки приложение захлебнулось.
Стек:
  • WEB:
    • Flask - основа приложения;
    • Werkzeug - WSGI составляющая Flask;
    • HTML + CSS + Bootstrap - статическая часть страниц;
    • JS + jQuery + Ajax - динамическая часть страниц;
    • JavaScript API - API сервиса Яндекс.Карты для отображения меток на карте;
    • Nominatim API - API сервиса Nominatim для преобразования геообъектов в координаты;
  • БД:
    • PostgreSQL - используемая СУБД;
    • SQLAlchemy - библиотека для подключения БД к приложению;
    ML:
    • Natasha - Python-фреймворк для NLP-задач на русском языке (здесь используется для обнаружения геообъектов).
